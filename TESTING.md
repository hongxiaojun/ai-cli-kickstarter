# AI CLI Kickstarter - 完整流程测试指南

## ✅ 项目完成情况

### 已完成的功能

1. ✅ **TUI 界面** - 带方向键选择的终端菜单
2. ✅ **默认国内用户** - 自动使用国内镜像
3. ✅ **环境依赖检测** - 启动时检测 Python、curl、curses
4. ✅ **网络环境检测** - 自动判断国内外网络
5. ✅ **安装后引导** - 生成使用指南并保存到桌面
6. ✅ **快速测试示例** - 提供测试 Prompt

### 项目结构

```
ai-cli-kickstarter-pro/
├── src/
│   ├── launcher.py        # 主入口（环境检测）
```

## 🧪 完整测试流程

### 步骤 1：运行安装器

```bash
cd /Users/add/ai-cli-kickstarter-pro
python3 run.py
```

**预期流程：**
1. 平台检测
2. Python 版本检查
3. curses 模块检查
4. curl 工具检查
5. 网络环境检测
6. **TUI 菜单界面**
   - 语言选择
   - Provider 选择
   - 网络状态显示
   - 确认安装
7. **安装完成**
   - 自动保存使用指南到桌面
   - 显示下一步引导

### 步骤 2：查看生成的指南

安装完成后，检查桌面是否有以下文件：
- `AI_CLI_使用指南.txt` - 完整使用指南
- `AI_CLI_快速参考.txt` - 快速参考卡片

### 步骤 3：启动 AI CLI 并测试

打开新终端窗口，根据安装的 Provider 启动 AI CLI：

```bash
# Qwen Code
qwen

# Kimi Code
kimi

# CodeBuddy CLI
codebuddy
```

### 步骤 4：粘贴使用规则

在 AI CLI 中粘贴以下规则（每次新会话）：

```
请把我当作计算机新手，始终用中文回复。
遇到我可能不懂的概念，先用通俗语言解释再继续。
动手改动之前：说明目标、说明风险、估计所需时间。
修改任何配置文件前先备份。
涉及管理员权限、删除、覆盖、付费或隐私的操作，先征得我的同意。
每完成一步都验证结果。
优先选择能解决问题的最简单方案。
```

### 步骤 5：测试创建文件夹（最终验证）

在 AI CLI 中输入：

```
帮我在桌面创建一个名为 '测试文件夹' 的文件夹
```

**预期结果：**
- AI CLI 理解你的意图
- 执行创建文件夹操作
- 告诉你操作结果
- 桌面上出现新文件夹

### 步骤 6：更多测试示例

继续测试其他操作：

## 📋 故障排查

### 问题 1：TUI 界面无响应

**原因**：curses 兼容性问题

**解决**：
- 程序会自动切换到文本备用模式
- 或使用 `python3 run.py` 重新运行

### 问题 2：无法启动 AI CLI

**原因**：PATH 环境变量未刷新

**解决**：
- 重启终端窗口
- 或运行 `source ~/.zshrc`（macOS/Linux）

### 问题 3：AI CLI 不响应

**原因**：AI CLI 可能需要登录或配置

**解决**：
- 检查 AI CLI 的错误提示
- 按提示完成登录或配置
- 查看桌面的使用指南文件
